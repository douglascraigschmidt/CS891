Movies Microservice (Assignment 1a)

The purpose of the first part of this two-part assignment is to acquaint you with developing synchronous microservices with Spring WebMVC. In particular, in this assignment you'll develop a synchronous microservice that uses Spring WebMVC to return List objects containing information about movies.

The code you'll write for this assignment does not have a GUI and runs as an Spring application with an automated test program. Please feel free to use this test program to help debug your solution.

The following resources may be helpful in completing this assignment:

  • An overview of Spring WebMVC appears here.

  • Professor Jules White's YouTube playlist has an excellent overview of Spring Web MVC.

Naturally, we'll cover this material in class, which will be available in my YouTube CS *891 playlist.

Assignment Structure and Functionality

The Movies microservice is packaged as a project using IntelliJ. This app is written in Java and demonstrates various Spring WebMVC features. For the purposes of Assignment 1a, however, you only need to be aware of the following directories:

  • recommender/movies, which contains the skeleton(s) you'll need to fill in, as described below.
  • movies/src/test, which is a set of unit tests that exercise the features you're implementing to evaluate whether you've correctly satisfied this assignment's requirements.

To compile this code you need to use the provided IntelliJ project. You can run this project by clicking the green arrow next to the "MoviesTest" dropbox menu in the IntelliJ IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers)!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the following files and folders:

  • MoviesController.java. You'll need to complete the "TODO" tasks in various fields/methods to define endpoint handler methods that return List objects containing information about movies.
  • MoviesService.java. You'll need to complete the "TODO" tasks in various fields/methods called by the MoviesController to return List objects containing information about movies.
  • Components.java. You'll need to complete the "TODO" tasks in factory methods (known as "Beans") that can be injected into Java classes using the Spring Autowired annotation to load a dataset containing movie titles and descriptive vectors.
  • MoviesSyncProxy.java. You'll need to complete the "TODO" tasks to implement proxies to various endpoints in the Movies microservice.

Your solution should strive to use no loops or if/else statements in this assignment and instead use Java sequential stream aggregate operations and functional programming features. Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments.

Unit Tests

The unit tests in the movies/src/test folder are provided to increase our collective confidence that your implementation is working as expected. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the unit tests to detect problems in your code.
