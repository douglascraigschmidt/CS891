Movies Microservice (Assignment 2a)

The purpose of the first part of this assignment is to deepen your understanding of how to develop multiple synchronous microservices with Spring WebMVC, the Eureka Discovery Service, the Spring Cloud Gateway, the Retrofit type-safe client API, and the Java Persistence API (JPA). In this assignment you'll apply JPA to create a database that stores information about movies and supports various SQL queries, as well as apply Retrofit to create several type-safe client APIs that access database and timer microservices via an API gateway.

The code you'll write for this assignment does not have a GUI and runs as an Spring application with an integration test program. Please feel free to use this test program to help debug your solution.

The following resources may be helpful in completing this assignment:

  • A tutorial on the Java Persistence API appears here.
  • A tutorial on Retrofit appears here.
  • I also recommend you use ChatGPT to help learn and code these various APIs.

Naturally, we'll cover this material in class, which will be available in my YouTube playlist.

Assignment Structure and Functionality

The Movies microservice is packaged as a project using IntelliJ. This app is written in Java and demonstrates various Spring WebMVC and Java structured concurrency features. For the purposes of Assignment 2a, however, you need to be aware of the following folders:

  • client, which contains a client test driver program containing proxy skeleton(s) you'll need to fill in, as described below.
  • database, which contains the database microservice skeleton(s) you'll need to fill in, as also described below.
You'll also need to tweak the synchronous movies-ex microservice that used Spring WebMVC from assignment 1c found in these folders:
  • movies-ex, which contains the skeleton(s) you'll need to fill in, as described below.
  • movies-ex/src/test, which is a set of unit tests that exercise the features you're implementing to evaluate whether you've correctly satisfied this assignment's requirements.

To compile this code you need to use the provided IntelliJ project. You can run this project by clicking the green arrow next to the "MoviesTest" dropdown menu in the IntelliJ IDE.

Program Description and "TODO" Tasks

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks (and do NOT delete the "TODO" markers or your submission will not be reviewed)!!! In particular, you'll need to finish implementing the "TODO" tasks for this assignment in the following files and folders:

  • client, where you'll need to complete the "TODO" tasks in various fields/methods to implement a type-safe client using Retrofit.
  • database, where you'll need to complete the "TODO" tasks in various fields/methods of the DatabaseService that use the Java Persistence API together with Java sequential and parallel streams to return Lists of objects containing information about movies.
  • You'll also need to complete the "TODO" tasks in factory methods in several ServerBeans.java and ClientBeans.java files that can be injected into Java classes using the Spring Autowired annotation.

As usual, your solution should strive to use few (if any) loops or if/else statements in this assignment and instead use Java functional programming features (including lambda expressions, method references, and sequential stream aggregate operations) and Java structured concurrency mechanisms. Your solution will be considered correct if it passes all the unit tests, is commented thoroughly, and addresses my review comments. Solutions that are not commented thoroughly will not be reviewed.
